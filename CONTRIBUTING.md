# 기여 가이드

Kytos OS에 기여해주셔서 감사합니다.
이 프로젝트는 살아있는 조직의 설계도를 함께 만들어가는 과정입니다.

---

## 시작 전에

### CLA 동의 필수

Pull Request를 제출하기 전에 [CLA.md](CLA.md)를 읽어주세요.
PR을 제출하면 CLA에 동의한 것으로 간주됩니다.

CLA가 필요한 이유:
기여자의 코드가 쌓이면 프로젝트의 라이센스를 전략적으로 조정할 수 없게 됩니다.
CLA는 프로젝트가 지속가능하게 운영되기 위한 최소한의 장치입니다.

---

## 기여할 수 있는 것들

```
스키마 개선      schema/ 내 JSON 파일 수정, 새 필드 제안
철학 문서        docs/ 내 문서 작성, 번역
사례 공유        examples/ 에 실제 사용 사례 추가
버그 리포트      이슈 등록
아이디어 제안    이슈 또는 Discussion
```

---

## 기여 과정

```
1. 이슈 먼저    — 큰 변경은 먼저 이슈로 논의합니다
2. 포크         — 레포를 포크합니다
3. 브랜치       — 목적에 맞는 브랜치를 만듭니다
4. 작업         — 변경사항을 만듭니다
5. PR 제출      — 변경 이유를 명확히 설명합니다
```

---

## 스키마 기여 원칙

- 필드는 최소한으로 유지합니다
- 하나의 스키마에 하나의 관심사만 담습니다
- 주석은 WHY만 씁니다. WHAT은 이름이 설명합니다
