# Kytos OS

> 영혼이 생동하는 사람들이 자연스럽게 모이고 흘러가며,  
> 그 흐름 자체가 생산이고 놀이이고 관계인 조직의 설계도.

---

## 이름

**Kytos** (κύτος) — 그리스어. 빈 용기, 세포.

생명의 최소단위. 경계가 있고 내부 구조가 있으며 외부와 선택적으로 교환한다.

Kytos OS 하나 = 하나의 조직 단위 = 하나의 노드.  
Kytos OS들이 연결될 때 → **Mycelium Network** 가 형성된다.

균사체(Mycelium)처럼 — 보이지 않는 망이 조직들을 연결하고,  
각 조직은 독립적이면서도 정보와 자원을 나눈다.

---

## 구조 원리

### Schema / Data 분리

```
Schema  (이 레포)       조직의 설계도. 오픈소스. 누구든 가져간다.
Data    (각 조직 소유)  실제 멤버, 프로젝트, 자원. 각자가 소유한다.
```

스키마는 포자처럼 가볍게 퍼진다.  
누구든 심으면, 그 땅의 조건에서 자기 방식으로 자란다.

### 시간 × 공간

```
수직 (시간계열)    씨앗 → 새싹 → 꽃 → 열매    성장과 단계
수평 (공간계열)    개인 ↔ 팀 ↔ 조직 ↔ 조직    확장과 연결
```

### AI Middle OS

AI는 명령을 실행하는 도구가 아니다.  
스키마를 해석하고, 의지를 실행으로 연결하는 중간 계층이다.

---

## 핵심 스키마

| 스키마 | 역할 |
|--------|------|
| `identity` | 조직의 존재 방식. 무엇으로 살아있는가. |
| `rhythm` | 흐름의 패턴. 어떻게 숨 쉬는가. |
| `relation` | 연결의 구조. 누구와 어떻게 연결되는가. |
| `resource` | 자원의 흐름. 무엇이 들어오고 나가는가. |
| `decision` | 결정의 방식. 어떻게 방향을 정하는가. |
| `growth` | 성장의 지표. 무르익음을 어떻게 아는가. |

---

## 설계 원칙

- 저비용, 저사양에서 작동한다
- 스키마는 가볍다. 데이터는 각자가 소유한다
- 시간(수직)과 공간(수평)을 모두 설계한다
- 확장이 용이한 가장 기본적인 골격에서 시작한다
- 사용하는 것만으로 자연스럽게 교육되고 성장한다
