# LIVING ORG OS PROJECT

> 영혼이 생동하는 사람들이 자연스럽게 모이고 흘러가며,  
> 그 흐름 자체가 생산이고 놀이이고 관계인 조직의 설계도.

이 프로젝트는 살아있는 조직의 운영체계(OS)를 오픈소스 스키마로 설계한다.  
누구든 이 스키마를 가져다 자신의 조직에 심을 수 있다.

---

## 설계 원칙

- 저비용, 저사양에서 작동한다
- 스키마는 가볍다. 데이터는 각자가 소유한다
- 시간(수직)과 공간(수평)을 모두 고려한다
- 확장이 용이한 가장 기본적인 골격에서 시작한다
- 사용하는 것만으로 자연스럽게 교육되고 성장한다
- 스키마는 AI 모델에 종속되지 않는다

## OS — Kytos (가제)

이 프로젝트가 만들고 있는 OS의 현재 이름. 아직 확정되지 않았다.

<details>
<summary>이름의 뜻과 선정 기준</summary>

### Kytos (κύτος)

그리스어. **빈 용기, 세포.**

생물학에서 세포(cell)를 뜻하는 모든 용어의 어근이다.  
`cyto-` (세포생물학 접두사), `-cyte` (세포 접미사) 모두 여기서 왔다.

### 왜 이 이름인가

이 OS를 쓰는 조직 하나 = 하나의 **세포(Kytos)**.

세포는 생명의 최소단위다.
- 세포막 — 경계가 있다. 내부와 외부가 나뉜다.
- 반투과성 — 경계가 있지만 선택적으로 교환한다.
- 소기관 — 내부에 구조가 있다.
- 독립적이면서 연결 가능 — 혼자서도 살아있고, 모이면 더 큰 것이 된다.

Kytos들이 연결될 때 → **Mycelium Network** 가 형성된다.  
균사체(Mycelium)처럼 — 보이지 않는 망이 조직들을 연결하고,  
각 조직은 독립적이면서도 정보와 자원을 나눈다.

### 선정 과정

`Mycelium → Spore → Hypha → Soma → Holobiont → Lumen → Theca → Cella → Kytos`  
생태학적 세계관을 확장하는 이름, 실제 존재하는 명사, 세포적 특성을 가진 것,  
라틴어·그리스어 어근 탐색을 거쳐 선정됐다.

## 구조 원리

### Schema / Data 분리

```
Schema  (이 레포)       조직의 설계도. 오픈소스. 누구든 가져간다.
Data    (각 조직 소유)  실제 멤버, 프로젝트, 자원. 각자가 소유한다.
```

스키마는 포자처럼 가볍게 퍼진다.  
누구든 심으면, 그 땅의 조건에서 자기 방식으로 자란다.

### 시간 × 공간

```
수직 (시간계열)    씨앗 → 새싹 → 꽃 → 열매    성장과 단계
수평 (공간계열)    개인 ↔ 팀 ↔ 조직 ↔ 조직    확장과 연결
```

### 스키마 포맷

**JSON**을 사용한다.  
기계 검증이 가능하고, 어떤 환경에서도 읽힌다.

## AI Middle OS

AI는 명령을 실행하는 도구가 아니다.  
스키마를 해석하고, 의지를 실행으로 연결하는 중간 계층이다.

**현재 Claude (Anthropic)로 작업하고 있다.**  
더 적합한 모델을 발견하면 교체할 수 있다.

이 스키마는 특정 AI 모델에 종속되지 않도록 설계한다.  
Claude, GPT, Gemini, 로컬 LLM 등 어떤 모델에서도 동일하게 작동하는 것을 목표로 한다.
