description: 세션 시작 — git pull, 마지막 세션 요약, 오늘 이어서 할 것 표시
---

일을 시작할 때 한 번만 실행합니다. `/task-start`보다 먼저 실행하세요.

---

**1단계: 환경 확인**

```bash
echo $KYTOS_DIR
```

비어 있으면 멈추고 안내합니다:
> `KYTOS_DIR` 환경변수가 설정되지 않았습니다.
> 터미널을 새로 열거나 `docs/setup.md`의 4단계를 확인해주세요.

---

**2단계: git pull**

```bash
cd $KYTOS_DIR && git pull 2>&1
```

결과에 따라:
- 정상 pull: "✓ 최신 상태로 업데이트됐습니다" 또는 "이미 최신 상태입니다"
- remote 없음 / 오류: "원격 저장소 없음 — 로컬 상태로 시작합니다" (멈추지 않음)

---

**3단계: 마지막 세션 확인**

```bash
cd $KYTOS_DIR && git log --oneline -5
```

최근 커밋 5개를 보여줍니다.

그리고 `individual/insights/` 에서 가장 최근 파일을 찾아 Read합니다:

```bash
ls -t $KYTOS_DIR/individual/insights/*.md | grep -v _template | head -1
```

해당 파일의 **"## 다음"** 섹션을 읽어 이어서 할 것을 파악합니다.

---

**4단계: 세션 브리핑**

다음 형식으로 출력합니다:

```
🌱 Kytos OS — 세션 시작
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
안녕하세요, {me.json의 name}님.

마지막 세션: {가장 최근 커밋 날짜}
  {최근 커밋 메시지 1~2개}

이어서 할 것:
  {마지막 insights 파일의 "## 다음" 섹션 내용}

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
준비됐으면 /task-start [작업 내용] 으로 시작하세요.
```

---

**참고**

- 세션 중에는 다시 실행할 필요 없습니다
- 작업 단위마다 `/task-start` → 작업 → `/task-end` 를 씁니다
- 하루 일과가 끝나면 마지막 `/task-end` 후 터미널을 닫으면 됩니다
